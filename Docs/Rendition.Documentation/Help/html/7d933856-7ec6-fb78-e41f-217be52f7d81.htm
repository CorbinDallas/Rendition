PaymentGatewayEventArgs Class See Also Send Feedback

[This is preliminary documentation and is subject to change.]

The PaymentGatewayEventArgs type exposes the following members.

Fields

  Name Description
Amount
IMPORTANT! Amount being charged.
BillToAddress
The bill to Address of the credit card.
Card
The credit card.
Message
The error message passed back to the client as to why the transaction has failed.
OrderNumber
Order number if any or _VERY_ short description. Should be unique (optional).
OrderSession
The session this order belongs to if any. Guid.Empty is used when no order session is present.
PreventDefault
IMPORTANT! Failure to set prevent default to true will cause the default credit card method to execute. So if you've subscribed to this event you should always set this boolean to true.
PurchaseOrder
Purchase order associated with this transaction (optional).
ShipToAddress
The ship to Address of the credit card.
SqlConnection
The SQL connection. Any access to the site database MUST use this connection.
SqlTransaction
The SQL transaction. Any access to the site database MUST use this transaction.
Success
IMPORTANT! Was the transaction a success? This is what you will need to set to tell the system the transaction was successfull. If the transaction fails you can set this to false to prevent further transactions.
