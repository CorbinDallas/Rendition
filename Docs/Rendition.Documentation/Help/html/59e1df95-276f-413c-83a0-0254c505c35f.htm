All About URLs.

A URL (Uniform Resource Locator) is an address on the internet. This is what you type into the address bar in your browser to go to a web site. For example http://www.google.com/ is a URL. Now let's examine the different parts of the URL.

Most of the time when people say "URL" they maybe should be saying URI. A URI (Uniform Resource Identifier) is a URL that contains a question mark "?". The question mark delineates the two parts of the URI.

The text to the left of the question mark is called the URL, while the text to the right of the question mark is called the URN (Uniform Resource Name).

Note

Throughout the topics in this documentation the URN will also be referred to as the QueryString [sic].

URL + URN = URI

URL

The URI is the URL plus the URN. Below is a URI. We'll take it apart and describe the pieces.

Copy 
http://www.mysite.com/list.aspx?category=plectrums&recordsPerPage=10
Remember, the text to the left of the question mark is called the URL. Below is the URL isolated from the rest of the URI.
Copy 
http://www.mysite.com/list.aspx
 The URL describes the location of a document on your web site. This URL is "pointing" to a file called list.aspx in the root directory.

Note

A directory is a folder and the root directory is the folder that all other folders are in.

The URL consists of three parts. The first part is called the protocol. The protocol describes how the file will be accessed. HTTP (Hyper Text Transfer Protocol) is the protocol used to access web pages. Other common protocols are FTP (File Transfer Protocol), HTTPS (Hyper Text Transfer Protocol Secure) and FILE (local file system).

Copy 
http://

The second part is the domain "www.mysite.com" is the domain. For more on domains see Domain Name System on Wikipedia

Copy 
www.mysite.com

The third part of the URL is the path. The path starts at your web site's root directory. Shown as "/" In our example the path is "/list.aspx". This path targets a file called list.aspx in the root directory.

Copy 
/list.aspx
To target a file in a subdirectory, the subdirectory name must be added to the path. In the following example the target is a file called "arrow.jpg" in the "img" directory.
Copy 
http://www.mysite.com/img/arrow.jpg
Note

Don't forget! A folder is a directory. Just another name for the same thing.

Caution

When a URL specifies a directory instead of a file, a default file within that directory is provided. For IIS servers, this default file is called "default.aspx". Each directory can have its own default file called "default.aspx". No default file will result in the error "Directory listing forbidden."

URN

The URN (Uniform Resource Name) describes the parameter or parameters being submitted to the URL. This is also called the QueryString [sic]. The URN begins after the first question mark in the URI.

The QueryString is a ampersand "&" separated collection of key/value pairs. The key/value pairs are themselves separated by a equal sign "=". Think of key/value pairs as a dictionary. The key is the name of the word and the value is the definition.

Consider the following URI.

Copy 
http://www.mysite.com/list.aspx?category=plectrums&recordsPerPage=10
The URN begins after the first question mark. The following is the URN isolated from the rest of the URI.
Copy 
category=plectrums&recordsPerPage=10
Note

The question mark is not part of the URI. It only delineates the beginning of the URI.

Next, we need to separate each key/value pair by separating each key/value pair. Each pair is separated by the ampersand sign "&". So in the example above there are two key/value pairs.

Key Value Pairs
category=plectrums
recordsPerPage=10
Next, each key/value pair is separated by the equal sign resulting in the following.
KeyValue
categoryplectrums
recordsPerPage10
Note

The equal sign is not part of the key or the value. It only delineates the key from the value.

Fragment Identifier

A fragment identifier is an optional piece of the URL that describes the location of an anchor on the document by the anchor's name attribute. The fragment comes after the URL and URN. A fragment always starts with the number sign "#" and is followed by the name of the anchor on the page to scroll to.

Note

An anchor is a link on a web page.

Below is an example of a URL with a fragment identifier.
Copy 
http://www.mysite.com/list.aspx?category=plectrums&recordsPerPage=10#top
 In the above example the fragment is called "top". When this page is visited, the web browser will look for an anchor with the name "top" and scroll to it.
Note

Fragments are not sent to the server, so they cannot be used in server side code (C#, etc.)

Absolute and Relative URIs

The difference between an absolute and relative URI is the presence of the protocol and domain of the URI. Relative URIs do not contain the protocol or domain.

A relative URI uses the protocol and domain of the page it is being displayed on. Below is an example of a relative URI.

Copy 
/list.aspx
And this is the same URI as an absolute URI.
Copy 
http://www.mysite.com/list.aspx
Caution

A relative URI makes a big assumption, that the missing portions of the URI can be found in the address bar of the browser. If this isn't the case, your relative URI will not work. Relative URIs should be avoided.

URL Encoding

Because the URI contains characters in it that describe directories, key/value pairs and other data there are some characters that cannot be used in the URI except in these predefined roles. For example, if one of your values in your URN (QueryString [sic]) contained a ampersand "&" it would be impossible to tell the value's ampersand from the key/value pair delineator.

To deal with this conflict, some characters should be "encoded" to let the server receiving the request know that the encoded data is not part of the URI's syntax, but rather the value of a key or key name in the URN.

Consider this URL.

Copy 
http://www.mysite.com/list.aspx?category=home&decor&recordsPerPage=10
The key "category" contains the value "home&decor" but it will appear to the server that the category is called "home" and that a new key/value pair begins with the "decor" and has no value. So to tell the server that the ampersand is part of the value we need to encode it. Below is an example of the correctly incoded URL.
Copy 
http://www.mysite.com/list.aspx?category=home%26decor&recordsPerPage=10
 Notice that the ampersand has been replaced by %27. This is the encoded version of the ampersand.

Here are some of the most common encoded characters. This is not an exhaustive list.

CharacterEncoded Character
Space%20
&%26
?%3F
=%3D
+%2B
%%25
#%23
@%40
Note

For more information on URL Encoding you can read this excellent tutorial at W3Schools: http://www.w3schools.com/tags/ref_urlencode.asp
