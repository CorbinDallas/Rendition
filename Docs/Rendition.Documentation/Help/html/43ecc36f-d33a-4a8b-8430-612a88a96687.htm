Rendition E-Commerce Framework
Installation

[This is preliminary documentation and is subject to change.]

Note

Rendition.dll requires IIS AspNetHostingPermissionLevel High or Full. Many inexpensive hosting providers do not offer this permission level on shared server plans.

Requirements

  • Windows Server 2003 (Standard, Web or Enterprise), Windows Vista Ultimate, Windows 7 Ultimate.
  • IIS 6, 7 or 7.5
  • SQL 2005, 2008, 2008 Express or LocalDB
  • IE9, Firefox, Chrome or Safari
  • Visual Studio 2010 (for development)

Installation Instructions

To install you'll need to download the following from Antidote 12.

  1. Rendition.dll E-Commerce library (Rendition.Core.dll).

  2. Rendition SQL Database Install Script (Install.sql).

Once you have the required files follow the following steps.

  1. Create a web site on your IIS server.

    Note

    When using IIS6 you must add the following line to the "Wildcard application maps" in the "Configuration..." dialog on the "Home Directory" tab of your new web site's Property dialog. The "Verify that file exists" checkbox should be unchecked.

    Copy 
    %systemroot%\microsoft.net\framework\v4.0.30319\aspnet_isapi.dll

  2. Copy Rendition.Core.dll into your new ASP.NET web site in the "bin" folder. If the "bin" folder does not exist, create it.

  3. Create a new database if you haven't done so already. The database's recovery model should be set to basic or simple if applicable.

  4. Open the SQL Install script (downloadable from Antidote 12) and read the instructions in the script. After the database is installed continue to the next step.

  5. Modify or create your Web.Config file in the root directory of your web site. Below are two example web.config files. The first example is for IIS6 and the second example is for IIS7. A full list of the application setting avaliable in Web.Config for Rendition can be found in the topic Web.Config.

    IIS 6 web.config

    Copy 
    <?xml version="1.0"?>
<configuration>
        <connectionStrings>
                <add name="sql" connectionString="Data Source=127.0.0.1,1433;Initial Catalog=MyDatabase;User ID=MyUserName;Pwd=MyPassword;Asynchronous Processing=true;MultipleActiveResultSets=true;"/>
        </connectionStrings>
        <appSettings>
                <add key="connectionName" value="sql"/>
                <add key="ElevatedSecurityUser" value="Rendition"/>
                <add key="ElevatedSecurityPassword" value="000000000000"/>
                <add key="ElevatedSecurityDomain" value="127.0.0.1"/>
        </appSettings>
        <system.web>
            <httpRuntime maxRequestLength="4194302" maxQueryStringLength="20000" requestValidationMode="2.0" requestPathInvalidCharacters="" />
                <httpModules>
                        <clear/>
                        <add type="Rendition.Main" name="Rendition.Main"/>
                </httpModules>
                <compilation debug="true"/>
  </system.web>
</configuration>

    IIS 7 web.config

    Copy 
    <?xml version="1.0"?>
<configuration>
        <connectionStrings>
                <add name="sql" connectionString="Data Source=127.0.0.1,1433;Initial Catalog=MyDatabase;User ID=MyUserName;Pwd=MyPassword;Asynchronous Processing=true;MultipleActiveResultSets=true;"/>
        </connectionStrings>
        <appSettings>
                <add key="connectionName" value="sql"/>
                <add key="ElevatedSecurityUser" value="Rendition"/>
                <add key="ElevatedSecurityPassword" value="000000000000"/>
                <add key="ElevatedSecurityDomain" value="127.0.0.1"/>
        </appSettings>
        <system.web>
            <httpRuntime maxRequestLength="4194302" maxQueryStringLength="20000" requestValidationMode="2.0" requestPathInvalidCharacters="" />
    <system.web>
        <system.webServer>
                <modules>
                        <add name="Rendition.main" type="Rendition.Main" />
                </modules>
                <handlers>
                        <add name="aspnet_isapi 32-bit" path="*" verb="*" modules="IsapiModule" scriptProcessor="%windir%\Microsoft.NET\Framework\v4.0.30319\aspnet_isapi.dll" resourceType="Unspecified" requireAccess="Script" preCondition="classicMode,runtimeVersionv4.0,bitness32" />
                        <add name="aspnet_isapi 64-bit" path="*" verb="*" modules="IsapiModule" scriptProcessor="%windir%\Microsoft.NET\Framework64\v4.0.30319\aspnet_isapi.dll" resourceType="Unspecified" requireAccess="Script" preCondition="classicMode,runtimeVersionv4.0,bitness64" />
                </handlers>
        </system.webServer>
</configuration>

  6. Rendition is now installed. To access the administrative tools navigate to the /admin subdirectory of your new web site. The default administrator's logon is:

    NamePassword
    adminpassword

  7. The first thing you should do before you start using Rendition is open the administrator's account by clicking "Edit Existing Account", click browse double click on the administrator's account (Admin) and change the password of the administrator then press save.

    Don't lose the password!

  8. Next click the "Settings" link and fill out as many of the fields as you feel like filling out. For a complete summary of what each field means visit the topic Settings.

    Don't lose the password!
