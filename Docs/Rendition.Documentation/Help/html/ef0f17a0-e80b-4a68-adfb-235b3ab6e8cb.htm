How to create a basic web site using Rendition E-Commerce library.

If you're not familar with editing ASP.NET Web Forms a great set of tutorials can be found at http://www.w3schools.com/aspnet/.

All examples use C#, HTML 5, jQuery 1.7 and CSS 3. A good understanding of ASP.NET Web Forms is a good idea before you get started using Rendition.

header.master

Header.master is a master page file. A master page file contains place holders that web forms (.aspx files) output to when the web form file is requested by a user. The master page file is defined in each web form.

This master page file in this example contains the header (top) and footer (bottom) content in any page that implements this master page file.

@ commands

The following commands take place at the very top of the page.

The following command defines this page as a master file.

Copy 
<%@ Master Language="C#" EnableViewState="false" %>
The next command includes a reference to Rendition.dll
Copy 
<%@ Import Namespace="Rendition" %>

Rendition.Site.Session

The following command will assign the current Rendition.Site.Session object to a new object called session. Then you can use this session object to access the user's cart, addresses, orders and preferences.

Get connected to the Rendition Session object.

Copy 
<%Site.Session session = Main.GetCurrentSession();%>

HTML 5 Master Page File

The following is just a reference on how to create your HTML 5 web page. This is an empty web page that contains "asp:ContentPlaceHolder" sections to demonstrate how to use a master page file. Web forms (.aspx files) that implement this master page file can inject HTML into the ContentPlaceHolders defined on the master page file. In the example .aspx files throughout this document these placeholder ids will be used.

The script "/admin/pub/mian.js" is generated by Rendition.dll and provides methods for loging on, off, ordering, PDF printing and more.

Note

Beginers should use this file to use Rendition. More advanced users can create their own JavaScripts that can interact with Rendition replacing this file with their own scripts.

The script "/js/jquery-1.4.2.min.js" should have been downloaded during the installation step. jQuery is a required for main.js to work.
Copy 
<!DOCTYPE html>
<html>
        <head>
                <title><asp:ContentPlaceHolder id="title" runat="server"></asp:ContentPlaceHolder></title>
                <meta name="description" content="<asp:ContentPlaceHolder id="metaDescription" runat="server"></asp:ContentPlaceHolder>">
                <script type="text/javascript" src="/admin/pub/main.js"></script>
                <asp:ContentPlaceHolder id="head" runat="server"></asp:ContentPlaceHolder>
        </head>
        <body>
                <asp:ContentPlaceHolder id="body" runat="server"></asp:ContentPlaceHolder>
                <asp:ContentPlaceHolder id="footer" runat="server"></asp:ContentPlaceHolder>
        </body>
</html>

Complete header.master

Copy 
<%@ Master Language="C#" EnableViewState="false" %>
<%@ Import Namespace="Rendition" %>
<%Site.Session session = Main.GetCurrentSession();%>
<!DOCTYPE html>
<html>
        <head>
                <title><asp:ContentPlaceHolder id="title" runat="server"></asp:ContentPlaceHolder></title>
                <meta name="description" content="<asp:ContentPlaceHolder id="metaDescription" runat="server"></asp:ContentPlaceHolder>">
                <script src="/admin/pub/main.js"></script>
                <script src="/js/jquery-1.4.2.min.js"></script>
                <asp:ContentPlaceHolder id="head" runat="server"></asp:ContentPlaceHolder>
        </head>
        <body>
                <asp:ContentPlaceHolder id="body" runat="server"></asp:ContentPlaceHolder>
                <asp:ContentPlaceHolder id="footer" runat="server"></asp:ContentPlaceHolder>
        </body>
</html>
